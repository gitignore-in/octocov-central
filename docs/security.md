# Security and Trust Boundaries

This repository delegates all artifact-to-SVG and artifact-to-Markdown conversion
to [k1LoW/octocov-action](https://github.com/k1LoW/octocov-action). The sections
below document what this repository trusts the upstream action to do and what
constraints consumers of the generated files should follow.

## Trusted artifact sources

The `central.reports.datastores` field lists `artifact://` URIs that the
workflow **fetches and processes** (input sources). Only URIs within the
`gitignore-in` organization are allowed. Adding a URI from an external owner
would redirect generated SVG and README content to that owner's artifact, which
may contain crafted content.

The `central.badges.datastores` field is semantically different: it is the
**output destination** where octocov writes the generated SVG badges. Its value
(`local://badges`) is a local path, not an artifact source, and it does not
affect the trust boundary described above. New member repositories are added to
`central.reports.datastores`, not to `central.badges.datastores`.

`.github/CODEOWNERS` designates `@kitsuyui` as a required reviewer for `.octocov.yml`
changes. This designation is advisory unless the `main` branch has protection rules
with "Require review from Code Owners" enabled.

## Source artifact availability

The Collect workflow checks every `.octocov.yml` `artifact://` URI before
running octocov. Each source repository must have at least one unexpired
artifact with the configured artifact name on the repository's default branch.
If any source artifact is missing, expired, inaccessible, or only available
from a non-default branch, the workflow fails before octocov can regenerate
`README.md` or SVG badges.

This makes artifact outages fail closed. The committed files under `badges/`
remain the last successful snapshot instead of being replaced by zero values or
partial output. The workflow also writes the selected source artifact metadata
to `badges/source-artifacts.json`, including the source repository, artifact
name, artifact id, creation time, expiration time, and source workflow run
branch/SHA when GitHub exposes that metadata. Use this file when checking
whether the committed badge snapshot is based on fresh source artifacts and
which source revision produced each artifact.

## Trust placed in upstream octocov

This repository trusts `k1LoW/octocov-action` to:

- XML-escape all data values before inserting them into SVG `<text>` nodes and
  attributes (boundary: artifact JSON → SVG XML).
- Encode repository names, metric labels, and numeric values correctly when
  rendering them in Markdown table cells (boundary: artifact data → Markdown).

If a future octocov release changes encoding behavior, generated badges and
`README.md` may reflect that change without any local code modification. Monitor
the [octocov changelog](https://github.com/k1LoW/octocov/releases) when
upgrading the pinned action SHA in `.github/workflows/central.yml`.

## README badge contract

The badge links committed in `README.md` are also a trust boundary. Each badge
URL must stay within the repository-local shape
`https://raw.githubusercontent.com/gitignore-in/octocov-central/main/badges/<owner>/<repo>/<metric>.svg`,
and the `<owner>`, `<repo>`, and `<metric>` path segments are compared after
percent-decoding and re-encoding each segment independently.

This keeps the README output tied to `.octocov.yml` rather than to any
free-form string that might appear in upstream output. The workflow checks this
contract with `scripts/check-readme-badge-contract.py` before octocov runs.

## Safe embedding of SVG badges

The SVG files under `badges/` are generated by octocov and committed to this
repository. They are safe to embed using an HTML `<img>` element:

```html
<img src="https://raw.githubusercontent.com/gitignore-in/octocov-central/main/badges/gitignore-in/gitignore-in/coverage.svg" alt="Coverage">
```

Do **not** embed them via `<object data="...">` or `<embed src="...">`. Those
elements allow SVG scripts to execute, whereas `<img>` treats the SVG as a
raster image and ignores any script content.

The badge copy snippets in `README.md` use Markdown image syntax
(`![alt](url)`), which renders as `<img>` in all standard Markdown renderers.
